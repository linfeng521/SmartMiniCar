# 模块化测试系统使用指南

## 📁 文件结构

```
BSP/
├── Test/
│   ├── test.h              # 测试接口头文件
│   ├── test.c              # 测试实现文件
│   ├── test_example.c      # 使用示例
│   └── README.md           # 本文档
├── Motor/
│   ├── motor.h
│   ├── motor.c
│   └── motor_test.md       # 电机测试指南
├── Encoder/
│   ├── encoder.h
│   └── encoder.c
└── HCSR04/
    ├── hcsr04.h
    └── hcsr04.c
```

## 🚀 快速开始

### 1. 添加文件到工程

在 Keil MDK 或 STM32CubeIDE 中：
1. 将 `test.c` 添加到工程
2. 将 `BSP/Test` 路径添加到 Include Paths

### 2. 在 main.c 中添加代码

```c
#include "test.h"

int main(void)
{
    // ... HAL 初始化 ...
    
    Motor_Init();
    Encoder_Init();
    HCSR04_Init();
    
    Test_Init();        // 初始化测试系统
    Test_PrintMenu();   // 显示测试菜单
    
    while (1)
    {
        // 方法1: 直接调用
        Test_Motor_Left();
        HAL_Delay(3000);
        
        // 方法2: 通过枚举
        Test_RunModule(TEST_MOTOR_RIGHT);
        HAL_Delay(3000);
    }
}
```

### 3. 通过串口控制（推荐）

发送单个字符命令：
- `0` - 测试左电机
- `1` - 测试右电机
- `2` - 测试前进
- `3` - 测试后退
- `4` - 测试左转
- `5` - 测试右转
- `6` - 测试原地左旋
- `7` - 测试原地右旋
- `8` - 测试左编码器
- `9` - 测试右编码器
- `a` - 测试双编码器
- `b` - 测试超声波
- `c` - 电机编码器同步测试
- `d` - 运行所有测试
- `m` - 显示菜单

## 📋 测试模块说明

### 电机测试

#### 1. 单电机测试 (`Test_Motor_Left` / `Test_Motor_Right`)
**目的**: 判断左右电机定义是否正确

**步骤**:
1. 调用 `Test_Motor_Left()`
2. 观察：应该是**左侧**电机转动
3. 如果是右侧转动，说明定义颠倒了

**预期输出**:
```
[测试] 左电机测试
左电机将以 50% 速度运行 2000 ms
请观察：应该是【左侧】电机转动
左电机测试完成
```

#### 2. 前进/后退测试 (`Test_Motor_Forward` / `Test_Motor_Backward`)
**目的**: 验证电机方向是否正确

**判断标准**:
- ✅ 前进：小车向前移动
- ✅ 后退：小车向后移动
- ❌ 如果相反，需要调整电机方向引脚

#### 3. 转向测试 (`Test_Motor_TurnLeft` / `Test_Motor_TurnRight`)
**目的**: 验证转向逻辑

**判断标准**:
- ✅ 左转：小车向左转弯（左轮慢，右轮快）
- ✅ 右转：小车向右转弯（右轮慢，左轮快）
- ❌ 如果相反，说明左右电机定义颠倒

#### 4. 原地旋转测试 (`Test_Motor_RotateLeft` / `Test_Motor_RotateRight`)
**目的**: 验证原地旋转功能

**判断标准**:
- ✅ 左旋：逆时针旋转（左轮后退，右轮前进）
- ✅ 右旋：顺时针旋转（左轮前进，右轮后退）

### 编码器测试

#### 1. 单编码器测试 (`Test_Encoder_Left` / `Test_Encoder_Right`)
**目的**: 验证编码器方向

**预期输出**:
```
[测试] 左编码器测试
左电机将前进 2000 ms，测量编码器计数
左编码器计数: 1234
方向: 前进
✓ 编码器方向正确（前进为正）
```

**判断标准**:
- ✅ 前进时计数为**正数**
- ❌ 前进时计数为**负数** → 编码器接线反了
- ❌ 计数为 0 → 检查编码器接线或 CubeMX 配置

#### 2. 双编码器同步测试 (`Test_Encoder_Both`)
**目的**: 检查两轮速度是否一致

**预期输出**:
```
[测试] 双编码器同步测试
小车将前进 2000 ms，测量两个编码器
左编码器: 1234
右编码器: 1256
差异: -22 (-1.8%)
✓ 两个编码器方向都正确
✓ 两轮速度基本一致
```

**判断标准**:
- ✅ 差异 < 10% → 速度基本一致
- ⚠️ 差异 > 10% → 可能需要 PID 校准

#### 3. 电机编码器同步测试 (`Test_Motor_Encoder_Sync`)
**目的**: 全面验证电机和编码器匹配

**测试流程**:
1. 前进 1 秒，记录编码器值
2. 后退 1 秒，记录编码器值
3. 分析结果

**预期输出**:
```
[测试] 电机编码器同步测试
1. 测试前进...
   左: 1234, 右: 1256
2. 测试后退...
   左: -1230, 右: -1250

结果分析:
✓ 电机和编码器方向完全匹配
```

### 传感器测试

#### HC-SR04 超声波测试 (`Test_HCSR04`)
**目的**: 验证超声波测距功能

**预期输出**:
```
[测试] HC-SR04 超声波测试
将连续测量 10 次距离
[1] 距离: 25.3 cm
[2] 距离: 25.5 cm
[3] 距离: 25.2 cm
...
```

**判断标准**:
- ✅ 能测量到距离（2-400cm）
- ✅ 数值稳定，波动小
- ❌ 显示 -1.0 → 超出测量范围
- ❌ 超时 → 检查接线或 CubeMX 配置

## 🔧 常见问题排查

### 问题1: 左右电机颠倒

**现象**: 调用 `Test_Motor_Left()` 时右侧电机转动

**解决方法**: 在 `motor.h` 中交换 A 和 B 的引脚定义
```c
// 交换前
#define MOTOR_AIN1_PIN GPIO_PIN_5
#define MOTOR_BIN1_PIN GPIO_PIN_3

// 交换后
#define MOTOR_AIN1_PIN GPIO_PIN_3
#define MOTOR_BIN1_PIN GPIO_PIN_5
```

### 问题2: 前进变后退

**现象**: 调用 `Test_Motor_Forward()` 时小车后退

**解决方法**: 在 `motor.c` 的 `Motor_SetDirection()` 中交换 SET 和 RESET

### 问题3: 编码器计数为负

**现象**: 前进时编码器显示负数

**解决方法**: 
1. 交换编码器 A/B 相接线
2. 或在 CubeMX 中修改编码器模式（TI1/TI2 互换）

### 问题4: 编码器无计数

**现象**: 电机转动但编码器计数为 0

**排查步骤**:
1. 检查编码器供电（通常 3.3V 或 5V）
2. 检查 A/B 相接线是否正确
3. 检查 CubeMX 中定时器配置（Encoder Mode）
4. 用示波器测量编码器输出波形

### 问题5: 超声波无数据

**现象**: 超声波测试一直超时

**排查步骤**:
1. 检查 TRIG 和 ECHO 接线
2. 检查 CubeMX 中 TIM4 配置（Input Capture）
3. 确认前方有障碍物（2-400cm 范围内）
4. 检查中断是否使能

## 💡 使用技巧

### 1. 快速诊断流程

```c
// 在 main 函数中添加
Test_Init();

// 步骤1: 测试左右电机
Test_Motor_Left();
HAL_Delay(3000);
Test_Motor_Right();
HAL_Delay(3000);

// 步骤2: 测试前进
Test_Motor_Forward();
HAL_Delay(3000);

// 步骤3: 测试编码器同步
Test_Motor_Encoder_Sync();

// 根据输出结果判断问题
```

### 2. 串口交互式测试（推荐）

```c
// 在串口接收回调中
void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart)
{
    if (huart->Instance == USART1) {
        char cmd = rx_buffer[0];
        
        if (cmd >= '0' && cmd <= '9') {
            Test_RunModule((Test_Module_TypeDef)(cmd - '0'));
        } else if (cmd == 'a' || cmd == 'A') {
            Test_RunModule(TEST_ENCODER_BOTH);
        }
        // ... 其他命令
        
        HAL_UART_Receive_IT(&huart1, rx_buffer, 1);
    }
}
```

### 3. 自动化测试

```c
// 开机自动运行所有测试
Test_Init();
Test_RunModule(TEST_ALL);
```
